# 一个简单的线程池
***
使用方法:
- SPSC单消费单生产无锁队列
- C++20协程
- io_uring(Linux 6.x)
***
线程池采用1分发线程，多个工作线程的组合，每个线程配备一个SPSC队列作为任务队列，有协程和io_uring混合，纯io_uring两种唤醒方式 

如果是纯io_uring阻塞唤醒速度有300-500W左右(由于数量级的不同)

如果是协程和io_uring混合阻塞唤醒有300-500W左右(由于数量级的不同)

如果是协程唤醒，也能达到400W，但是协程唤醒的话因为其挂起的性质导致线程会在挂起以后直接执行下一个指令，需要保证线程不会提前终止，可以尝试一个线程当两个用，这样的话就需要把代码修改一下，现在主要还是混合唤醒

以上测试都是都是执行一个原子加法任务

**这个仅仅只是作为协程初学的使用练习，实际上纯io_uring的稳定性和实用性会更高**
